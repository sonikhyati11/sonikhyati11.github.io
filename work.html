The Life cycle of a Component

At Niyo Design system we took an atomic design approach to create components, by creating/focusing on atoms and going ahead with molecules and then organisms and templates.Documentation is as important as designing the components. Documenting everything including the resuable components journey of where to use, where not to use, different states of it and the nomenclatures. Each component shoud serve a clear purpose, multiple components should not serve the same purpose Taking refereances from existing systems is also a part of the process. We did research on many existing design systems took inspiration according to our product's needs.

We made sure to understand the thought processes and decisions in order to keep improving the system and to make it scalable on long term.

Journey

First we decided the sequence of components to be created. Then we created the Figma project and different artboards for different categories of components. We also created a sandbox for experimentation on each component, trying out auto layout etc.

We conducted User testing and got it reviewed by stakeholders before publishing it. Now Individual designers can drag and drop the Figma component into their design. If the design system team needs to make a change to the component, they can make it once and deploy.

Handover

We maintained handover notes which are the instructions for developers on writing the code for the component. The handover document begins with the anatomy of the component, then includes its variations.

PS: A design system is never done; it needs continuous improvement.
